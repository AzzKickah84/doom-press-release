  • Full installer: DOOM-Setup-3.0.5.exe (all WADs from the build folder).

Version 3.0.3 14 June 2026

New

  • Classic DOS startup console — while the game loads, a DOS-style text window shows the original startup banner and subsystem init lines (V_Init, Z_Init: Init zone memory allocation daemon, R_Init: Init DOOM refresh daemon, and the rest). Closes when the game window opens; stays open with -devparm or -console; disable with -noconsole.

Improved

  • DOOM Forge 3.1 — live sector fill tinted from floor textures; undo/redo (32 steps, Ctrl+Z/Y); texture search (Ctrl+F); thing rotation (Q/E and mouse wheel); upper/mid/lower wall texture slots; sector and vertex delete; automatic two-sided lines on shared boundaries; map lump rename (F2); unsaved-changes prompt; Test in DOOM uses the correct -warp from the lump name.

Version 3.0.2 12 June 2026

Fixed

  • Launcher game options — duplicate Windows control IDs broke No Monsters, Respawn, and Fast Monsters checkboxes; episode/map spin controls now use separate IDs so launch arguments match the UI.

Improved

  • Launcher layout — larger footer gap and shorter resolution dropdown list reduce overlap with footer buttons when combos are open.

Version 2.0.13 12 June 2026

Version 2.0.12 12 June 2026

New

  • Customizable display resolution — eight window-size presets from 320×200 through 2560×1600. Options → Display → Resolution replaces the old scale slider. Saved as display_resolution in default.cfg; legacy window_scale values migrate automatically.
  • Texture filter (upscale quality) — Options → Display → Texture Filter: Sharp (pixel-perfect nearest-neighbor) or Smooth (linear filtering when the 320×200 framebuffer is stretched). Works best with OpenGL or Direct3D presentation. Saved as scale_quality.
  • Launcher display settings — Display Resolution and Texture Filter dropdowns on the Game tab. Choices persist via launcher_display.txt and apply on launch.

Fixed

  • Mind Control Device — converted allies are no longer invincible to the player; you can kill them with normal weapons if needed. Stale hit-target state cleared before each mind blast.
  • Main menu keyboard — arrow keys, Enter, Esc, and hotkeys work again alongside mouse-driven menus. SDL input is polled every frame instead of only during tic catch-up.
  • Graphics detail toggle — Options → Setup → Graphics (high/low detail) calls R_SetViewSize again instead of being disabled.
  • Press release visitor counter — retro counter starts at 0000001 instead of a fake 1993-themed seed.

Improved

  • Command-line flags: -res N (resolution preset 0–7), -smooth / -linear, -sharp / -nearest. Legacy -2, -3, -4 still map to presets.
  • Title/demo screen no longer swallows keyboard input when the menu is already open.

Version 2.0.9 13 June 2026

Version 1.10.19 12 June 2026

Fixed

  • Sound effects distorted or too loud: SDL mixer now adds pre-scaled lookup values directly (matching original i_sound.c) and uses correct 16-bit sample stepping for pitch.
  • No music on modern Windows: replaced broken MCI MIDI playback with OPL3 synthesis via libADLMIDI, using the GENMIDI lump from your IWAD and mixed into the same SDL audio stream.
  • I_InitMusic is now called at startup so the music subsystem initializes properly.

Version 1.10.17 12 June 2026

Fixed

  • Keyboard controls not working when default.cfg contained DOS scan codes instead of DOOM key codes.
  • Sound effects too loud or distorted: corrected stereo volume lookup in the SDL mixer.

Version 1.10.16 12 June 2026

Fixed

  • Launcher crash 0xC0000005 when starting a game: maptexture_t now matches the WAD byte layout (packed struct, 4-byte masked field, 4-byte obsolete columndirectory) so textures load correctly on 64-bit.
  • Texture pointer tables allocated with sizeof(*ptr) instead of 4 bytes per entry on 64-bit Windows.
Version 1.10.15 12 June 2026

Fixed

  • Crash 0xC0000005 when starting a game from the launcher: maptexture_t used a 2-byte obsolete field where the WAD format has 4 bytes, so textures after index ~125 were read with width/patchcount 0, corrupting the memory heap during R_InitTextures.
  • 64-bit config loader: chat macro and string defaults now store full pointers instead of truncated 32-bit values (m_misc.c).
Version 1.10.14 12 June 2026

Fixed

  • Crash 0xC0000005 right after startup: same 64-bit pointer truncation bug in R_InitTranslationTables as the earlier COLORMAP fix.
  • Crash when loading a level: 64-bit fixes in zone memory allocator (Z_Free / Z_ChangeTag).
  • Increased default zone memory to 16 MB on Windows.
Version 1.10.13 12 June 2026

Version 1.10.0 12 June 2026

Initial Windows release

  • Port of original linuxdoom 1.10 source to Windows using SDL2.
  • Graphical launcher for IWAD/PWAD selection, skill level, episode, and map.
  • Inno Setup installer with Start Menu and optional desktop shortcut.
  • Command-line -iwad support for custom WAD paths.
  • SDL2 video rendering, digital sound, keyboard, and mouse input.
  • Winsock networking support for multiplayer (-net).
  • Game options: No Monsters, Respawn, and Fast Monsters via launcher or command line.

Components (initial)

  • doom.exe — game engine
  • doomlauncher.exe — pre-game configuration GUI
  • SDL2.dll — SDL2 runtime library

Later releases add doomforge.exe (map editor, v1.10.34+), doom_extra.wad (Mind Control weapon, v1.10.31+), and radar (v1.10.30+). Version 3.0.4 is the current release.
